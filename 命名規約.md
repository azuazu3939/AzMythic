# AzMythic 命名規約

## 基本方針

- 外部から直接参照される mmid は、役割がパスと名前の両方で分かるように英語の責務語で統一する。
- 表示名やアイテム id の日本語はそのまま使ってよいが、内部参照名は英語の役割語を優先する。
- 同一 feature では、`public` と内部 helper で接頭辞を揃え、末尾の役割語だけを変える。

## mmid の形

### 公開入口

外部から直接呼ばれる公開スキルは、次の形を基準にする。

```text
Azuriter_<Layer>_<Group>_<Feature>_<Entry>
```

例:

- `Azuriter_Item_Weapon_Melee_Tomahawk_Use`
- `Azuriter_Item_Weapon_Ranged_EmpuriaBow_Use`
- `Azuriter_Mob_Boss_Wedler_Loop`
- `Azuriter_Mob_Boss_Wedler_Attack1`
- `Azuriter_Mob_Boss_Wedler_DropHigh`

### 内部 helper

内部 helper は公開入口と同じ prefix を保ち、末尾で責務だけを分ける。

```text
Azuriter_<Layer>_<Group>_<Feature>_<Entry>_<Role>
```

例:

- `Azuriter_Mob_Boss_Wedler_Attack1_Process`
- `Azuriter_Mob_Boss_Wedler_Attack1_Effect`
- `Azuriter_Item_Weapon_Ranged_FireBulletBow_ShotCharge`
- `Azuriter_Item_Weapon_Ranged_EmpuriaBow_BurstRelay`

### Aura / Tag / Var

Aura / Tag / Var は layer と kind を前に出す。

```text
Azuriter_<Layer>_<Kind>_<Feature><State>
```

例:

- `Azuriter_Core_Tag_AttackBusy`
- `Azuriter_Item_Aura_EmpuriaBowBurst`
- `Azuriter_Item_Var_RangedShotCounter`
- `Azuriter_Mob_Var_FeldosAttack4ExtendCount`

## 役割語の使い分け

- `Use`: アイテムや装備セットから直接呼ばれる入口
- `Loop`: ボスや常時挙動の再帰ループ
- `Attack<N>`: ボスや敵の攻撃入口
- `DropHigh` / `DropSpawn`: ドロップ処理
- `Effect`: 演出専用
- `Process`: 主処理
- `Run` / `Repeat` / `Check`: 中間段階
- `Shot` / `Burst`: 遠隔武器の派生処理
- `L1` / `R1`: 左右派生がゲーム上の概念として存在する時だけ使う

## ファイル配置

- 単一責務の feature は `main.yml`
- 入口と helper が分かれる feature は `public.yml` と責務別ファイル
- ボスは `public.yml`, `attack-*.yml`, `drop.yml`, `effect.yml`, `utility.yml`
- 遠隔武器は `public.yml`, `shot.yml`, `burst.yml`, `charge.yml`
- 1 ファイルに複数定義を置く場合も、同一責務の範囲に留める

## 本文の書き味

### 見出しコメント

- 公開入口ファイルの先頭には、1 行の日本語コメントで責務を書く
- 例: `# エンピュリアボウの発動入口。`

### セクション順

原則として次の順番で書く。

1. `Cooldown`
2. `Conditions`
3. `TargetConditions`
4. `Skills`

`Skills` の中では、次の流れを基本にする。

1. コスト消費とクールダウン
2. state / var の更新
3. 予告演出や移動
4. 本ダメージや主効果
5. 後処理と再帰

## 運用メモ

- `items/*.yml`, `equipment-sets.yml`, `mobs/*.yml` から直接呼ばれる名前を変えた時は、参照先も同時に更新する。
- mmid を変えた時は、必要に応じて `F:\\AzuriteProject\\Azuriter` 側の参照も検索して確認する。
